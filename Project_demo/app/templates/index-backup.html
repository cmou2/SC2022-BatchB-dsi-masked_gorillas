California Housing Prices Prediction

Predicting housing value in California based on a variety of factors.

See our work below

What does the project do?

The dataset we found on Kaggle contains 20,000 entries from a census about housing prices during the 1990's. It contains the information of the location, median house age, total rooms, and the median house value. Our goal is to find patterns among these features and predict California housing prices.

    Machine Learning


    Linear Regression: Explanation

    In the most simple words, Linear Regression is the supervised Machine Learning model in which the model finds the best fit linear line between the independent and dependent variable i.e it finds the linear relationship between the dependent and independent variable.

    Assumptions:

  • Linearity: It states that the dependent variable Y should be linearly related to independent variables.
  • Normality: The X and Y variables should be normally distributed.
  • Homoscedasticity: The variance of the error terms should be constant i.e the spread of residuals should be constant for all values of X. This assumption can be checked by plotting a residual plot. If the assumption is violated then the points will form a funnel shape otherwise they will be constant.
  • Independence/No Multicollinearity: The variables should be independent of each other i.e no correlation should be there between the independent variables. To check the assumption, we can use a correlation matrix or VIF score. If the VIF score is greater than 5 then the variables are highly correlated.
  • The error terms should be normally distributed.
  • No Autocorrelation: The error terms should be independent of each other.

    • Linear Regression: RMSE

    RMSE is an acronym for Root Mean Square Error, which is the square root of value obtained from Mean Square Error function. Using RMSE, we can easily plot a difference between the estimated and actual values of a parameter of the model. By this, we can clearly judge the efficiency of the model. In this model calculated an unusual RMSE, but this is to be expected, as our data covers all of California and is not specific enough when applying to exact cities.

    Support Vector Machine Regressor: Explanation

    Support Vector Machine, or SVR, is unlike other regression models in that it doesn't try to minimize the error between the real and predicted value. Instead, it tries to find the best fit line within a threshold value and the distance between the hyperplane and boundary line, and then uses that to make a prediction.

    Assumptions:

  • The dependent variable is numerical rather than categorical.
  • Unlike SLR, whose results depend on Gauss-Markov assumptions, the output model from SVR does not depend on distributions of the underlying dependent and independent variables.
  • The SVR technique depends on kernel functions.

    • Support Vector Machine Regressor: Results

    SVR used grid search to calculate the Mean Squared Error of our data set. The output number is risk function corresponding to the predicted value of the squared error loss. We use the number in our code to measure how close a regression line is to different variables. SVR involves two parameters: the regularization parameter (often referred to as C) and the error sensitivity parameter (often referred to as ϵ). In this model we got a regularization parameter of 1000 and an sensitivity parameter of 0.01.

    K Nearest Neighbors: Explanation

    K Nearest Neighbor, or KNN, is a non-parametric regression method that, in an intuitive manner, approximates the association between independent variables and the continuous outcome by averaging the observations in the same neighborhood.

    Assumptions:

  • Similar things exist in close proximity. In other words, similar things are near to each other. The KNN algorithm hinges on this assumption being true enough for the algorithm to be useful. KNN captures the idea of similarity (sometimes called distance, proximity, or closeness) with calculating the distance between points on a graph.

    • K Nearest Neighbors: RMSE

    The size of the neighborhood needs to be set by the analyst or can be chosen using cross-validation (we will see this later) to select the size that minimizes the mean-squared error. Our K Nearest Neighbor for our data set is 11, which represents the ideal neighborhood to use in our code to decide which factors relate most. The RMSE corresponds to the square root of the average difference between the observed known outcome values and the predicted values.In this model calculated an unusual RMSE, but this is to be expected, as our data covers all of California and is not specific enough when applying to exact cities.

    Random Forest: Explanation

  • Ensemble learning is the process of using multiple models, trained over the same data, averaging the results of each model ultimately finding a more powerful predictive/classification result.
  • Bootstrapping is the process of randomly sampling subsets of a data set over a given number of iterations and a given number of variables.

    • The bootstrapping Random Forest algorithm combines ensemble learning methods with the decision tree framework to create several randomly drawn decision trees from the data, averaging the results to output a new result that often leads to strong predictions and classifications.

    Assumptions:

  • The input data is continuous, and the target variable is discrete
  • The input data contains multiple variables, and each variable has only one level
  • There are no missing values in the input data
  • The data is distributed normally.

    • Random Forest: RMSE

    Mean squared error (MSE) is the average of the summation of the squared difference between the actual output value and the predicted output value. Our goal is to reduce the MSE as much as possible.The root mean squared error (RMSE) is just simply the square root of the MSE. This model calculated an unusual RMSE of 81684, but this is to be expected, as our data covers all of California and is not specific enough when applying to exact cities.

    Lasso Regressor: Explanation


    Lasso regression analysis is a shrinkage and variable selection method for linear regression models. The goal of lasso regression is to obtain the subset of predictors that minimizes prediction error for a quantitative response variable.

    Assumptions:

  • Lasso regression uses shrinkage, where the data values are shrunk towards a central point such as the mean value.
  • The lasso regression allows you to shrink or regularize these coefficients to avoid overfitting and make them work better on different datasets.
  • Lasso is a supervised regularization method used in machine learning.
  • In statistics and machine learning, lasso (least absolute shrinkage and selection operator; also Lasso or LASSO) is a regression analysis method that performs both variable selection and regularization in order to enhance the prediction accuracy and interpretability of the resulting statistical model.
